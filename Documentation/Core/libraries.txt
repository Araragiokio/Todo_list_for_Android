# TodoApp — Libraries Used

## Overview

TodoApp is built using **React Native with Expo** and relies on several libraries to handle navigation, animations, storage, notifications, and other core functionality.

These libraries provide ready-made solutions for common mobile app features and allow the application to be developed efficiently without reinventing standard components.

All libraries used in the project are compatible with the **Expo environment**.

---

# Core Framework

## React Native

Purpose:
Provides the foundation for building the mobile application using JavaScript and native components.

Responsibilities:

* Rendering UI components
* Managing component lifecycle
* Handling user interaction
* Running the application logic

---

## Expo

Purpose:
Expo simplifies React Native development by providing tools, APIs, and libraries that work without requiring native configuration.

Responsibilities:

* Project setup and development environment
* Access to device APIs
* Running the app through Expo Go
* Simplified package management

---

# Navigation

## @react-navigation/native

Purpose:
Provides the core navigation system used for moving between screens.

Responsibilities:

* Managing navigation state
* Handling screen transitions
* Supporting navigation actions

---

## @react-navigation/bottom-tabs

Purpose:
Creates the bottom tab navigation used as the main navigation structure.

Responsibilities:

* Rendering the tab bar
* Managing tab switching
* Displaying icons and labels for each tab

---

## react-native-screens

Purpose:
Improves navigation performance by optimizing screen rendering.

Responsibilities:

* Efficient screen transitions
* Reduced memory usage

---

## react-native-safe-area-context

Purpose:
Ensures UI elements respect safe areas on devices with notches, rounded corners, or system UI overlays.

Responsibilities:

* Managing safe layout margins
* Preventing UI overlap with device elements

---

# Animations

## lottie-react-native

Purpose:
Used to display high-quality animations created in Lottie format.

Responsibilities:

* Splash screen animations
* Task completion sparkle animation
* Confetti celebration animation
* Sakura background animation

Animations are stored in:

assets/animations/

---

## react-native-reanimated

Purpose:
Provides advanced animation capabilities for smooth UI transitions.

Responsibilities:

* Performance-optimized animations
* Gesture-based animation support

---

# Notifications

## expo-notifications

Purpose:
Handles scheduling and displaying notifications.

Responsibilities:

* Task reminders
* Deadline alerts
* Future push notification support

This system will be expanded during later development phases.

---

# Storage

## @react-native-async-storage/async-storage

Purpose:
Provides persistent local storage for the application.

Responsibilities:

* Saving tasks
* Loading tasks
* Updating task data
* Storing theme preferences
* Storing custom categories

AsyncStorage ensures that data remains available even after the app is closed.

---

# Audio

## expo-av

Purpose:
Provides support for playing sound effects within the app.

Responsibilities:

* Task completion sound
* Notification sound
* Feedback audio

Audio feedback enhances the user experience by reinforcing actions.

## expo-image

# Summary

The libraries used in TodoApp enable the following core capabilities:
