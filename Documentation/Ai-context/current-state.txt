# Navigation State

The app uses bottom tab navigation for the main product areas.

Final tab order:

Home | Add Task | Reminders | Analytics | Profile

Settings is no longer a bottom tab.

Settings is accessed through a **top-right icon on the Profile screen**.

This change improves UX clarity and keeps primary navigation focused on high-frequency actions.

---

# Screen Responsibilities

Home
Displays tasks and the main dashboard.

Add Task
Creates and manages new tasks.

Reminders
Shows scheduled and overdue tasks.

Analytics
Displays productivity insights and planned visualizations including pie chart, bar chart, heatmap, and smart insights.

Profile
Displays user identity, personal data, streaks, totals, future achievements, and settings access.

---

# Architecture Direction

The app now follows an explicit separation of concerns between identity and performance:

Profile = Identity
Analytics = Performance

Analytics features should not be placed inside Profile.

Reason:

* Keeps screens focused
* Improves scalability
* Prevents Profile from becoming overloaded
* Makes future analytics expansion easier

---

# Analytics State

The current analytics work is a UI skeleton only.

Planned analytics features:

Pie Chart
Completed tasks grouped by category with Week / Month / Year filters.

Bar Chart
Tasks completed per day in a weekly view.

Heatmap
Long-term consistency across days.

Insights
Text-based feedback generated from user productivity data.

Real data processing is planned for the next phase.

# Developer Notes

* Avoid hardcoded layout hacks like `marginTop` for spacing fixes
* Use proper safe area handling
* Keep navigation minimal
* Do not mix identity, settings, and analytics responsibilities into one screen
